## Spacing Scale

Declared values (must be multiples of 4). This phase lives inside the right sidebar — a narrow panel (~280px). All spacing stays in the sm–lg range to respect that constraint.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, badge padding, chip inner gap |
| sm | 8px | Between document card meta rows, between filter chips |
| md | 16px | Card internal padding, section padding within tab content |
| lg | 24px | Between section groups (e.g., knowledge base list → autonomy settings) |
| xl | 32px | Empty state vertical centering padding |
| 2xl | 48px | Not used in this phase (panel is too narrow) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Drop zone target height: 80px minimum (usability, not a spacing token)
- Touch targets (delete button, dial buttons): 32px minimum height, 32px minimum width
- Document card list item: 12px (3) top/bottom padding — a 3px exception to the 8pt grid, matching the existing `ApprovalItem` at `py-3` (12px)

Source: measured from `ApprovalItem.tsx` (`px-3 py-3`), `SidebarTabBar.tsx` (`px-2 py-2`), `EmptyState.tsx` (`py-12 px-6`)

## Color

All values reference existing Hatchin CSS custom properties. No new colors introduced in this phase.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--hatchin-surface)` / `var(--hatchin-panel)` | Tab content background, drag-drop zone background |
| Secondary (30%) | `var(--hatchin-surface-elevated)` / `var(--hatchin-card)` | Document cards, settings panel cards, work output cards |
| Accent (10%) | `var(--hatchin-blue)` (#6C82FF) | Specific reserved uses — see below |
| Destructive | `var(--destructive)` / red-500 | Delete document confirmation only |

Accent (`--hatchin-blue`) is reserved for:
1. Active autonomy dial position highlight (selected segment background or border)
2. Primary "Upload Document" button fill
3. Document type badge on PDF files (per established `hatchin-blue` badge pattern)
4. Focus ring on the drop zone when drag is active (2px solid `rgba(108,130,255,0.6)`)

Accent is NOT used for: card backgrounds, section headers, icons in non-interactive states, or general decoration.

Secondary semantic color:
- `var(--hatchin-green)` (hsl(158, 66%, 47%)): autonomy dial "Autonomous" position indicator — this signals the highest-trust state, green communicates permission/go
- `var(--hatchin-orange)` (hsl(25, 100%, 60%)): document type badge for DOCX files, and the "Observe" dial position dot (low autonomy / watchful state)

Type badge color assignment (pre-decided so executor doesn't have to guess):
| File type | Badge color |
|-----------|-------------|
| PDF | `--hatchin-blue` |
| DOCX | `--hatchin-orange` |
| TXT | muted (`--hatchin-text-muted` on `--hatchin-surface`) |
| MD | `--hatchin-green` |

Source: `index.css` custom property definitions, `SidebarTabBar.tsx` badge patterns, `ApprovalItem.tsx` border-left color semantics.

- Active drag state: border changes to `2px solid var(--hatchin-blue)`, background `rgba(108,130,255,0.06)`

- Error state: border turns `2px solid var(--destructive)`, label shows error text (see Copywriting)

### DocumentCard
- Surface: `bg-[var(--hatchin-surface-elevated)]`, border `var(--hatchin-border-subtle)`, border-radius 10px, padding 12px
- Left side: file type icon (FileText/FileCode/File from lucide, 16px) + type Badge
- Title: filename truncated at 1 line, 14px semibold, `--hatchin-text-bright`
- Meta row: upload date formatted as "Mar 26" (short format), `--hatchin-text-muted`, 12px
- Delete button: TrashIcon (Lucide, 14px), placed at card trailing edge, `--hatchin-text-muted`, becomes `red-400` on hover, touch target 32x32px, no confirmation dialog (undo-able via re-upload — see Copywriting for toast)
- Skeleton loading state: use existing `.skeleton-shimmer` class from `index.css`
- Animation: `motion.div` with `initial={{ opacity:0, y:6 }} animate={{ opacity:1, y:0 }}` (matches ApprovalItem pattern)

### AutonomySettingsPanel
- Section header: "Autonomy Settings", 11px, semibold, `--hatchin-text-muted`, uppercase letter-spacing 0.05em
- Enabled toggle row: Switch (shadcn `ui/switch.tsx`), label "Autonomous execution", 13px semibold `--hatchin-text`, sub-label "Hatches can work in the background" 11px muted
- Inactivity trigger row (only shown when enabled): "Auto-start after" label + Select (shadcn `ui/select.tsx`) with options: "30 minutes", "1 hour", "2 hours", "4 hours". Default: 2 hours.
- Autonomy Dial: 4-segment horizontal segmented control (not a Radix Slider — this is a discrete selector)
  - Segments: Observe | Propose | Confirm | Autonomous
  - Active segment: background `var(--hatchin-blue)`, text white, border-radius inherits from container
  - Inactive segments: background transparent, text `--hatchin-text-muted`
  - Container: rounded-lg border `var(--hatchin-border-subtle)`, background `var(--hatchin-surface)`, padding 2px
  - "Autonomous" position: active background is `var(--hatchin-green)` (not blue) to signal full permission granted
  - Segmented control height: 32px, full width of panel minus 16px side padding
  - Below dial: a one-line description that updates per position (12px muted, italics — see Copywriting)

### WorkOutputSection
- Section header: "Work Outputs", 11px semibold muted uppercase
- Only shown when at least 1 completed background deliverable exists
- Empty: no section rendered (no empty state for this sub-section — saves vertical space)
- WorkOutputCard: expandable via Collapsible (shadcn `ui/collapsible.tsx`)
  - Collapsed: agent avatar (24px circle initial), agent name + task title (12px, truncated to 1 line), timestamp (11px muted), ChevronDown icon

  - Background: `var(--hatchin-surface)`, border `var(--hatchin-border-subtle)`, border-radius 8px

## Interaction Contracts

### File Upload Flow (BRAIN-01)
1. User drags file over DocumentUploadZone → zone enters active drag state (blue border)
2. User drops file → client validates type (PDF/DOCX/TXT/MD) and size (≤10MB) before any network call
3. If validation fails → show inline error inside zone (no toast), zone border turns red
4. If validation passes → show loading state inside zone, POST to `/api/projects/:id/brain/documents`
5. On success → zone resets to idle, new DocumentCard animates in at top of list, toast "Document added to brain"
6. On server error → zone shows error state, offers retry (see Copywriting)
7. "Browse files" click → triggers hidden `<input type="file">` without any confirmation

### Document Delete (BRAIN-02)
- Single click on TrashIcon → immediate DELETE request, no confirmation dialog
- Optimistic removal: card exits with `motion` exit animation before server confirms
- On server error → card animates back in, toast "Couldn't remove document. Try again."
- No undo mechanism (document must be re-uploaded)

### Autonomy Toggle (BRAIN-03)
- Toggle on → inactivity trigger row slides in below with height animation (Framer Motion `height: 'auto'`)
- Toggle off → inactivity trigger row slides out, dial remains visible but dims to 50% opacity
- All settings persist via PATCH `/api/projects/:id` on change (debounced 800ms)
- No save button — auto-save with flash-save border animation on the settings panel

### Autonomy Dial (BRAIN-03)
- Click a segment → segment highlights immediately (optimistic), PATCH sent after 800ms debounce
- No animation between segments beyond color change — do NOT use a sliding pill (matches existing segmented control patterns in the codebase)

### Work Output Expand (BRAIN-04)
- Click collapsed card → expand with Collapsible open animation
- Expanded state persists during session (not across refresh)
- Only one card expanded at a time (accordion behavior)

## Accessibility Contract

- DocumentUploadZone: `role="button"`, `aria-label="Upload document — drag and drop or click to browse"`, keyboard-activatable (Enter/Space triggers file picker)
- DocumentCard delete button: `aria-label="Delete {filename}"` (dynamic)
- Autonomy toggle Switch: already accessible via shadcn Switch (Radix CheckedState)
- Autonomy dial segments: `role="radiogroup"` on container, `role="radio"` on each segment, `aria-checked` per segment
- Inactivity trigger Select: already accessible via shadcn Select (Radix)
- WorkOutputCard Collapsible: `aria-expanded` managed by shadcn Collapsible
- Section headers: use `<h3>` with visually styled as 11px uppercase label — preserve heading hierarchy

## Wire-Up Notes for Executor

These are integration points the executor needs — pre-identified from existing code patterns:

1. **BrainDocsTab slot in RightSidebar**: The tab panel already has CSS-hide infrastructure from Phase 11. Add `<div style={{ display: activeTab === 'brain' ? 'flex' : 'none' }} className="flex-col flex-1 min-h-0">` wrapping `<BrainDocsTab projectId={activeProject?.id} />`.

2. **Project brain PATCH**: Use existing `PATCH /api/projects/:id` endpoint. The `brain` field is JSONB: `{ documents: [], sharedMemory: string }`. Autonomy settings will be stored in `projects.executionRules` JSONB (already exists per schema).

3. **File upload endpoint**: Use existing `POST /api/projects/:id/brain/documents` (already defined in routes per CLAUDE.md API reference). Server uses `multer v2.0.2` — not v1.

4. **Autonomy settings persistence**: Store in `projects.executionRules` JSONB as `{ autonomyEnabled: boolean, autonomyLevel: 'observe'|'propose'|'confirm'|'autonomous', inactivityTriggerMinutes: number }`. PATCH via existing project update endpoint. No DB migration needed — fits existing JSONB column.

5. **Work outputs query**: Fetch from existing `GET /api/tasks?projectId=...` filtering for `status === 'completed'` and `metadata.hasOutput === true`. No new endpoint required.
