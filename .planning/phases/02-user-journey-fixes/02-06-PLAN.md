Two fixes in this task — both inside CenterPanel.tsx.

**Fix A — UX-07: agentRole in streaming placeholder**

In the `streaming_started` WS handler, when creating the `streamingMessage` object, look up the responding agent's role from `activeProjectAgents` and inject it into `metadata.agentRole`. The `message.agentId` field is present in the streaming_started payload.

Find the `streamingMessage` object literal (the one with `metadata: { isStreaming: true }`) and change its `metadata` field to:

```typescript
metadata: {
  isStreaming: true,
  agentRole: (() => {
    const agent = activeProjectAgents.find(a => a.id === (message.agentId || ''));
    return agent?.role ?? null;
  })()
}
```

This ensures the streaming placeholder has the correct role from the moment it is created, so MessageBubble renders with the right color immediately without waiting for `streaming_completed`.

**Fix B — DATA-04: Guard agentRole backfill useEffect against empty agent list**

In the `useEffect` that transforms `apiMessages` (the one with deps `[apiMessages, currentChatContext, activeProjectAgents]`), add an early-return guard at the top of the effect body to prevent running when `activeProjectAgents` is empty for the current project:

```typescript
useEffect(() => {
  if (apiMessages && currentChatContext && Array.isArray(apiMessages)) {
    // Guard: if activeProject has no agents loaded yet, defer transform to avoid
    // rendering all messages with null agentRole (green flash).
    // The effect will re-run when activeProjectAgents populates.
    if (activeProjectAgents.length === 0) return;

    // ... existing transform logic unchanged ...
  }
}, [apiMessages, currentChatContext, activeProjectAgents]);
```

Note: This guard is safe because `activeProjectAgents` is in the dep array — React will re-run the effect once agents load. Messages will not render at all until agents are available, which avoids the green flash. The loading skeleton (already present) handles the empty state.

Do NOT modify the transform logic inside the effect — only add the `if (activeProjectAgents.length === 0) return;` guard line at the top.
