**Step 2 — Wrap the team agent list**

Replace the current:

```typescript
{/* Team Agents */}

  <div className="ml-7 space-y-0.5">

  </div>
```

With:

```typescript
{/* Team Agents */}
<AnimatePresence initial={false}>
  {isTeamExpanded && (
    <motion.div
      key={`agents-${team.id}`}
      initial={{ height: 0, opacity: 0 }}
      animate={{ height: 'auto', opacity: 1 }}
      exit={{ height: 0, opacity: 0 }}
      transition={{ duration: 0.18, ease: 'easeInOut' }}
      className="overflow-hidden"
    >
      <div className="ml-7 space-y-0.5">
        {teamAgents.map(agent => { ... })}
      </div>
    </motion.div>
  )}
</AnimatePresence>
```

Key implementation notes:
- `initial={false}` on AnimatePresence prevents the expand animation from firing on first page load when the team is already expanded — only user interactions trigger the animation.
- `key={`agents-${team.id}`}` — unique per team so AnimatePresence can track enter/exit correctly across multiple teams.
- `className="overflow-hidden"` on the motion.div — required for height animation; without it, content overflows during the transition.
- `transition={{ duration: 0.18 }}` — matches existing transition durations used elsewhere in ProjectTree (`transition-all duration-200`).
- The inner `<div className="ml-7 space-y-0.5">` and all its children remain unchanged — only the outer wrapper changes.

Apply this pattern to every `{isTeamExpanded && ...}` instance in the teams map loop. There should be exactly one instance per team (in the `{projectTeams.map(team => { ... })}` block).
