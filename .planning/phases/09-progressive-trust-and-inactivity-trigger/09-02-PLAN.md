**Change 2: server/autonomy/background/backgroundRunner.ts** — Modify `runAutonomousExecutionCycle` (starting at line 211) with two changes:

(a) Replace the lastUserActivityAt detection block (lines 225-229) to use `project.lastSeenAt` instead of last message:

Replace:
```typescript
let lastUserActivityAt: Date | null = null;
try {
  const recentMessages = await _storage.getMessagesByConversation(`project:${project.id}`, { limit: 1 });
  if (recentMessages[0]?.createdAt) lastUserActivityAt = new Date(recentMessages[0].createdAt);
} catch { /* non-critical */ }
```

With:
```typescript
const lastUserActivityAt: Date | null = project.lastSeenAt
  ? new Date(project.lastSeenAt)
  : null;
```

This is more accurate — lastSeenAt tracks when the user actually opened the project (set by join_conversation in chat.ts), not when the last message was sent (which could be from an agent).

(b) After the `if (!trigger.shouldExecute) continue;` line (line 240), add the inactivityAutonomyEnabled gate:

```typescript
if (!trigger.shouldExecute) continue;

// EXEC-04: Gate inactivity triggers on per-project flag
if (
  trigger.reason === 'inactivity' &&
  !project.executionRules?.inactivityAutonomyEnabled
) {
  devLog(`[BackgroundRunner] Skipping inactivity trigger for project ${project.id} — inactivityAutonomyEnabled is false`);
  continue;
}
```

This ensures:
- Explicit triggers (user said "go ahead") still work regardless of inactivityAutonomyEnabled
- Inactivity triggers only fire when the project has opted in
- The check happens BEFORE queueTaskExecution, so no wasted job queue writes

Also add `triggerSource` to the autonomy event logged by the cron (for observability). After the `queueTaskExecution` call inside the for-loop (line 251), add a log:

```typescript
await queueTaskExecution({ taskId, projectId: project.id, agentId: agent.id });
```

```typescript
// Test 10: Inactivity trigger fires after 3 hours with pending tasks
const threeHoursAgo = new Date(Date.now() - 3 * 60 * 60 * 1000);

  autonomyEnabled: true,
  lastUserActivityAt: threeHoursAgo,
});
assert(result10.shouldExecute === true, `Test 10: shouldExecute should be true, got ${result10.shouldExecute}`);
assert(result10.reason === "inactivity", `Test 10: reason should be 'inactivity', got ${result10.reason}`);

// Test 11: Under inactivity threshold (30 min) — no trigger
```
