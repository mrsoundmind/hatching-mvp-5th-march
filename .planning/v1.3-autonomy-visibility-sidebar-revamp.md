# v1.3 — Autonomy Visibility & Right Sidebar Revamp

**Started:** 2026-03-24
**Goal:** Make the powerful autonomy backend (v1.1) visible and controllable from the frontend. Revamp the right sidebar from static brain metadata into a live autonomy command center with activity feed, approvals hub, and redesigned brain/docs.

## The Problem

v1.1 built background execution, agent handoffs, safety gates, trust scoring, peer review, and deliberation traces. But the frontend shows almost none of it:
- Right sidebar is static textarea fields (brain, direction, culture) — feels gimmicky
- No visibility into what agents are doing in the background
- Agent handoffs appear as plain chat messages — no visual chain
- Approval cards only appear inline in chat — easy to miss
- No file upload for project brain documents
- No autonomy controls or settings UI
- No way to see deliverables produced by background agents

## Scope (15 items)

### Sidebar Restructure
1. **Right sidebar revamp** — tabbed layout: Activity / Brain & Docs / Approvals
2. **Live autonomy progress feed** — real-time event stream in sidebar
3. **Autonomy stats card** — summary: tasks completed, handoffs, cost spent
4. **Activity feed filters** — filter by event type, agent, or time range
5. **Empty state design** — compelling empty states when no data yet

6. **Handoff visualization** — chat announcement cards + sidebar timeline chain

### Approvals & Tasks
10. **Pending approvals hub** — dedicated sidebar section with actionable items
11. **Task pipeline view** — visual: queued -> assigned -> in-progress -> review -> done

### Brain & Settings
12. **Project brain redesign** — PDF/doc upload + cleaner knowledge base display
13. **Autonomy settings UI** — inactivity toggle, cost cap visibility, autonomy level control
14. **Work output viewer** — browse/review deliverables from background agents

## Architecture Decisions

| Decision | Rationale |
|----------|-----------|
| Decompose RightSidebar.tsx into tab shell + child components | 850-line monolith too large; tabs need independent state |
| CustomEvent bridge from CenterPanel to sidebar | Single WS connection stays in CenterPanel; sidebar listens via events (existing pattern) |
| multer + pdf-parse for file upload | No external storage (S3) needed at MVP scale; base64 in brain.documents JSONB |
| No DB migrations | All data fits existing JSONB columns + autonomy_events table |
| Wrap-then-restructure sidebar | Tabs first, then refactor content — avoids breaking existing functionality |

## Backend Data Already Available

The backend is rich — this milestone is primarily frontend work:
- 50+ autonomy event types in `autonomy_events` table
- `GET /api/autonomy/events` — event stream (needs projectId filter)
- `GET /api/autonomy/traces` — deliberation traces
- Trust scoring in `agents.personality.trustMeta`
- Handoff chains with cycle detection (max 4 hops)
- Safety thresholds: <0.35 auto, 0.35-0.59 peer review, >=0.60 user approval
- Configurable policies in `server/autonomy/config/policies.ts`
- WS events defined but not all wired to UI yet
